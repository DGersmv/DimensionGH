# Проверка меню Info227 в Grasshopper

## Важно: Как работают категории в Grasshopper

В Grasshopper категории (меню) создаются **автоматически**, когда есть хотя бы один компонент в этой категории. 

## Шаги для проверки:

1. **Соберите проект** в Visual Studio (Build → Build Solution)
   - Убедитесь, что сборка прошла без ошибок

2. **Проверьте, что файл создан:**
   - Должен быть файл: `bin\Debug\DimensionGH_Gh.dll` или `bin\Release\DimensionGH_Gh.dll`

3. **Скопируйте плагин в Grasshopper:**
   - Скопируйте `.dll` файл в: `%AppData%\Grasshopper\Libraries\`
   - **Переименуйте** файл в: `DimensionGH_Gh.gha` (важно: расширение должно быть `.gha`, а не `.dll`)

4. **Перезапустите Rhino и Grasshopper полностью**

5. **Проверьте в Grasshopper:**
   - Откройте Grasshopper
   - В панели компонентов найдите категорию **"Info227"**
   - Должен быть компонент **"Dim_Connect"** в подкатегории **"Connection"**

## Если меню не появляется:

1. Проверьте, что файл `.gha` находится в правильной папке:
   ```
   %AppData%\Grasshopper\Libraries\DimensionGH_Gh.gha
   ```

2. Проверьте логи Grasshopper:
   - File → Special Folders → User Object Folder
   - Ищите файлы с ошибками

3. Убедитесь, что:
   - Файл переименован в `.gha` (не `.dll`)
   - Rhino и Grasshopper полностью перезапущены
   - Нет конфликтов с другими плагинами

4. Попробуйте удалить старый файл и скопировать заново

## Текущая настройка:

- **Категория:** "Info227"
- **Символ категории:** 'I'
- **Компонент:** Dim_Connect
- **Подкатегория:** Connection

Категория появится автоматически, когда компонент будет загружен в Grasshopper.
