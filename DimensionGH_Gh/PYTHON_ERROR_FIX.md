# Исправление ошибки GhPython

## Ошибка: "There must be a unique PythonAssemblyInfo-derived type in __grasshopper__module"

Эта ошибка **НЕ связана с нашим плагином** - она от плагина GhPython (Python для Grasshopper).

### Возможные причины:

1. **GhPython пытается загрузить Python скрипты из папки Libraries**
   - Проверьте папку: `%APPDATA%\Grasshopper\Libraries\`
   - Убедитесь, что там нет Python файлов (`.py`) которые могут конфликтовать

2. **Конфликт с другими плагинами**
   - Другие плагины могут иметь проблемы с GhPython

3. **Наш плагин загружается правильно, но ошибка от другого плагина**

### Решение:

1. **Проверьте, что наш плагин работает:**
   - Откройте Grasshopper
   - Найдите категорию "Info227"
   - Если она есть - наш плагин работает правильно!

2. **Игнорируйте ошибку GhPython (если наш плагин работает):**
   - Эта ошибка не влияет на наш C# плагин
   - Нажмите OK и продолжайте работу

3. **Если хотите исправить ошибку GhPython:**
   - Проверьте папку `%APPDATA%\Grasshopper\Libraries\`
   - Удалите или переименуйте проблемные Python файлы
   - Или обновите/переустановите GhPython

### Проверка нашего плагина:

Наш плагин правильно зарегистрирован если:
- ✅ Категория "Info227" видна в Grasshopper
- ✅ Компонент "Dim_Connect" доступен
- ✅ Нет ошибок при загрузке нашего плагина (только GhPython ошибка)

### Важно:

Наш плагин - это **C# плагин**, а не Python. Ошибка GhPython не должна влиять на его работу.
